# NeetCode Offline TUI

Study NeetCode 150 on a plane ‚Äî no internet required after setup.

## Quick Start

```bash
# 1. Create and activate virtual environment
uv venv
source .venv/bin/activate

# 2. Install dependencies
uv pip install -e .
uv pip install yt-dlp   # optional, only needed for video downloads

# 3. Fetch problem data while online (takes ~2 min)
python prepare.py

# 4. Launch the TUI
python run.py
```

## Editor Setup

The TUI opens your system `$EDITOR` for coding. Set it in your shell rc:

```bash
# ~/.bashrc or ~/.zshrc
export EDITOR=nvim    # or vim, nano, etc.
```

Vim-style navigation is provided via the external editor ‚Äî there is no
in-TUI editor. The TUI suspends while you code, then resumes when you quit
the editor.

## Test Workflow

1. Press `e` on a problem to open your editor with a starter template.
2. Write your `Solution` class.
3. Quit the editor.
4. Press `r` to run tests ‚Äî hand-curated pass/fail results appear on the
   right pane for the first ~50 problems. For others, LeetCode example
   inputs are shown without expected output.

Example output:
```
‚úì PASS  [1] inputs=[[2, 7, 11, 15], 9]
‚úì PASS  [2] inputs=[[3, 2, 4], 6]
‚úì PASS  [3] inputs=[[3, 3], 6]
Passed 3/3
```

## Spaced Repetition (SRS)

Reviews are scheduled with [FSRS](https://github.com/open-spaced-repetition/py-fsrs).
When every test passes you're prompted to rate your recall (Again / Hard / Good /
Easy); you can also rate any time with `a`. The list screen marks reviewed
problems with `‚úì` and due problems with `‚Üª`, and the SRS screen (`x`) shows every
card's state, next due date, and stability.

- Press `[` / `]` on the SRS screen to tune desired retention (default 90%).
- Opening a *due* problem from the SRS screen deletes your saved attempt so you
  re-solve it from scratch.
- SRS state lives in `data/srs.json`, with a rolling set of timestamped backups
  in `data/srs_backups/`.

### AI Critique (optional)

Press `c` on a problem to get a critique of your attempt. Set either
`ANTHROPIC_API_KEY` or `DEEPSEEK_API_KEY` (in your shell or `.env.local`);
without one the feature stays offline. Anthropic takes priority when both are set.

## Video Downloads

Videos are **never downloaded automatically**. Two ways to get them:

**Bulk (before your flight):**
```bash
python prepare.py --videos   # downloads all 150 at 480p (~10‚Äì15 GB)
```

**On-demand (from the TUI):**
Press `v` on any problem. If the video is missing you'll be asked to confirm
the download (~30 MB). This runs `yt-dlp` in a background thread so the UI
stays responsive.

Videos are stored in `data/videos/<slug>.mp4` and played with `mpv` (falls
back to `vlc`, `xdg-open`, `open`).

## Custom Interview Prep Questions

Beyond the NeetCode 150, the list ends with an **Interview Prep (Custom)**
section of broader Python programming questions (output prediction, builtin
reimplementation, concurrency design, data wrangling). Each ships with a full
editorial (press `s`) and a runnable reference solution.

These cards work just like NeetCode problems ‚Äî edit (`e`), show the editorial
(`s`), rate your recall (`a`), and they feed the same SRS scheduler.

They are seeded in `data-back/` and are **never fetched online** ‚Äî `prepare.py`
only touches the NeetCode 150, so re-running it leaves these untouched.

To add your own:

1. Add the slug to `CUSTOM_TOPICS` in `src/problem_list.py`.
2. Add a problem entry (keyed by slug) to `data-back/problems.json` ‚Äî set
   `content_text` to the question and `python_snippet` to a starter template.
3. Add an editorial at `data-back/editorials/<slug>.txt` (use ```python fences
   for code ‚Äî they're syntax-highlighted) and, optionally, a reference solution
   at `data-back/solutions/<slug>.py`.
4. Copy the new files into your live `data/` dir (or re-seed it from
   `data-back/`).

## Troubleshooting

**LeetCode 403:** The fallback API (`alfa-leetcode-api.onrender.com`) kicks
in automatically. You may see slower fetches.

**`mpv` not found:**
```bash
sudo apt install mpv      # Debian/Ubuntu
sudo pacman -S mpv        # Arch
brew install mpv          # macOS
```

**Textual rendering glitches over SSH:**
```bash
export TERM=xterm-256color
```

**Import errors:** Make sure you activated the venv with
`source .venv/bin/activate` before running.
